Laboratorio IA local
Guia de laboratorio IA local para Mac

Montar ecosistemas de IA para aprender con control

El objetivo no es tener el modelo mas potente ni ganar una carrera de productividad. Es aprender a pensar con herramientas: medir recursos, decidir que delegas, entender que ocurre en tu maquina y construir proyectos propios con paciencia.

La espera tambien forma parte del experimento: sirve para ajustar expectativas, no para apagar el cerebro.

Antes de instalar nada

La IA no sustituye tu voluntad

Un modelo puede proponer, resumir o ejecutar pasos, pero tu decides el objetivo.

La lentitud enseña

Cuando un modelo tarda, pregunta que esperas, que podria fallar y como sabras si el resultado sirve.

Git es memoria del proyecto

Git guarda versiones y ayuda a revisar lo que cambia antes de aceptarlo.

Avisos antes de elegir ruta

Seguridad y privacidad

No pegues contraseñas, claves API, tokens ni datos privados en herramientas que no entiendes. Local no significa automaticamente seguro.

Comandos de ejemplo

Los comandos son una ruta posible para practicar. En macOS muchas herramientas tambien se instalan desde la web oficial del proyecto: Git, Docker Desktop, LM Studio, Warp, Ollama, etc.

Elige un itinerario de pruebas

Comandos por ruta

Base comun: herramientas del Mac

Instala primero lo necesario para que las otras rutas tengan suelo.

previo
Observa

Comprueba que tienes herramientas de desarrollo, Git, Docker y Warp.

Piensa

Git es una libreta de cambios para revisar y volver atras.

Instalar herramientas base
xcode-select --install
brew update
brew install git
brew install --cask docker
brew install --cask warp
brew install --cask visual-studio-code
Configurar identidad Git
git config --global user.name "Tu Nombre"
git config --global user.email "tu-email@example.com"
git config --global init.defaultBranch main

Ruta 2: OpenCode + VS Code + Git

Para administrar proyectos con versionado y revisar cambios antes de aceptarlos.

media
Objetivo

Crear un proyecto pequeno, abrirlo en VS Code y revisar el diff con Git.

Versionado

Haz commit antes de pedir cambios al agente.

Instalar o verificar OpenCode
brew install sst/tap/opencode
opencode --version
opencode auth list
Crear proyecto versionado
mkdir laboratorio-git-ia
cd laboratorio-git-ia
git init
printf "# Laboratorio Git + IA\n" > README.md
git add README.md
git commit -m "Inicio del laboratorio"
code .
opencode .

Ruta 4: Ollama + llama3.2 + Open WebUI

Modelo local, API y una interfaz web en Docker.

completa
Objetivo

Entender las capas: Ollama ejecuta el modelo, Open WebUI es la interfaz y Docker contiene la app.

Recursos

Observa CPU, memoria, temperatura y disco.

Instalar Ollama y bajar llama3.2
brew install ollama
ollama serve
ollama pull llama3.2
ollama run llama3.2
Levantar Open WebUI con Docker
docker run -d \
  -p 3000:8080 \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main

open http://localhost:3000
